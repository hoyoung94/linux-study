## 실습한 명령어

| 명령어 | 용도 |
| --- | --- |

| `sudo journalctl -f` | 시스템 로그 실시간 확인 |
| `man` | 명령어 매뉴얼 보기 |

## 헷갈렸던 점

처음에는 `syslog` 파일로 시스템 로그를 확인하려 했지만, 스터디원 중 일부는 해당 파일이 존재하지 않았다. 그래서 `journalctl` 명령어로 로그를 확인했는데, 서로 출력 내용이 달라 의문이 생겼다.

찾아보니 두 도구의 성격이 달랐다.

- **`journalctl`**: 시스템이 켜지자마자 발생하는 원천 데이터를 모두 보여줌
- **`syslog`**: 그중 일부를 골라 사람이 읽기 좋게 다듬은 요약본

즉, `journalctl`이 더 풍부한 정보를 담고 있고, `syslog`는 정제된 버전이라는 점을 이해하게 되었다.
